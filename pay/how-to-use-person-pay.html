# 个人免签支付收款解决方案

随着手机支付的流行，越来越多的服务商开始接入移动支付，使用最为广泛的是支付宝支付和微信支付，然而这些支付方式都要去企业资质才可以申请，对于个人商户而已，没有注册公司没有营业执照无法直接使用相关API。 当然这难不倒聪明的国人，出了很多种个人支付的方式，总结有以下三种：

方式 门槛 费率 优点 缺点
对接第三方api 低 高 由第三方提供收付款接口，支持实时通知 费率高，需要手动提现，有“跑路”风险
挂机监听到款通知 高 无 实时监听支付宝、微信到账通知，无费率 需要安装特定APP、长期运行手机、安装配置麻烦、易风控
人工审核 低 无 实时到账、无需提现、资金安全，免费 需要人工审核

对接第三方api是最简单的，可是费用也是最高的；利用挂机应用貌似不错，可是配置和开发复杂，且容易被风控，无法稳定运行；人工审核虽然无法实现收款回调，但总的来看是最折中的方案。

# 支付流程

  1. 商户上传收款码 商户事先生成若干收款码，包括固定金额的收款码和不设金额的通用收款码（后称固码和通用码），解析二维码内容并保存到数据库中。
  2. 用户下单 用户填写订单，提交订单到服务器。服务器根据金额和支付方式从数据库中匹配收款码（默认寻找固码，找不到匹配通用码），成功匹配后发送邮件通知商户（邮件内容带上订单信息）。
  3. 用户支付 下单用户通过扫码或唤起APP的形式支付。
  4. 管理员审核并通过订单 用户支付成功后，商户根据收到的邮件和到账通知判断是否通过审核，信息正确后，点击邮件上的审核按钮，修改订单状态。
  5. 页面显示支付成功 支付页面通过轮询或websocket的方式定时询问服务端，查询订单状态，订单通过后显示支付成功。

需要注意的是，支付宝生成收款码有个限制，大概是每天可生成20张，其次留意二维码解析结果，这将是唤起APP的关键。
