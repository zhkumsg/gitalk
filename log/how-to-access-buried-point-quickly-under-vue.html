    # 在vue下如何快速接入埋点

    在前端监控三部曲之数据埋点一文中阐述了多个埋点平台和数据埋点的大致流程

    在常规开发中，我们更多关注的是pv和事件，对操作中的一些实时信息需求没有很高（如输入框内容），所以大部分站长都会采用简单的 百度统计、友盟统计 或 Google Analytics。

    这几个平台接入都很简单，分三步：

    1. 注册账号

    2. 创建应用

    3. 引入sdk

    但是简单却不代表一定好，没采用一个技术我们都需要考虑接入难度、接入成本。接入难度已经没有问题了，引入一个sdk文件即可，但是接入成本却要认真考虑了，对于一个大型应用，简单采集pv没有意义不是特别大，我们更关心的是用户操作习惯，即事件分析，但是，事件分析却又不可避免侵入代码

    有没有两全齐美的办法？既可以采集足够多的行为信息，又可以做到尽可能少浸入逻辑

    还真的有，我们采用全埋点 + 声明埋点结合的方式，通过在特定元素上绑定事件，当事件被触发时自动上报数据

    需要注意以下几点

    1. 注入什么数据

2. 如何遍历dom

3. 如何绑定事件

4. 如何上报数据

    这几个问题在vue下通过指令能完美解决，既可实现任意数据类型注入、又无需关注dom，按需使用，注册事件也是动态绑定，数据上报直接掉底层方法即可

    下面正式介绍vue-trackevent的使用

    # 安 装

    $ npm install vue-trackevent --save

    # 使 用

    <template>
    <button v-trackevent="['hello','world']">DEBUG</button>
</template>

    // main.js
import Vue from 'vue';
import VTrackevent from 'vue-trackevent';
 
Vue.use(VTrackevent, {
    callback(data, e) {
        console.log(data, e);
    },
    debug: false,
});

    # 属 性

    属性 说明
    init 初始化时调用的函数，只调用一次，主要是配合第三方 SDK
    callback 元素触发后的回调，data 为绑定的数据，e 为事件源对象
    debug 是否处于调试环境，默认 false

    # 数 据

    v-trackevent 支持绑定 可被克隆的 Javascript 对象 ，如 Number、String、Array、Object、Boolean

    默认元素触发方式为点击 click，事件绑定方式为追加

    如果需要更改触发方式，如mouseover，则需要按下述要求修改传入的值

    v-trackevent="{event:'mouseover',data:'hello world'}"

    即需要传入{event,data}的 object 数据

    # 点 击 触 发

    <template>
    <button v-trackevent="{event:'click',data:['hello','world']}">DEBUG</button>
</template>

    等同于

    <template>
    <button v-trackevent="['hello','world']">DEBUG</button>
</template>

    # 鼠标进入触发

    <template>
    <button v-trackevent="{event:'mouseenter',data:['hello','world']}">DEBUG</button>
</template>

    # 文本改变触发

    <template>
    <input type="text" v-trackevent="{'event':'change',data:['hello world']}" />
</template>

    // main.js
import Vue from 'vue';
import VTrackevent from 'vue-trackevent';
 
Vue.use(VTrackevent, {
    init() {
        window._hmt = window._hmt || []; // 挂载到window上
        const hm = document.createElement('script');
        hm.src = `https://hm.baidu.com/hm.js?xxx`; // 百度统计应用id
        const s = document.getElementsByTagName('script')[0];
        s.parentNode.insertBefore(hm, s);
    },
    callback(data) {
        if (data instanceof Array && '_hmt' in window) {
            window._hmt.push(['_trackEvent'].concat(data)); // 上报信息
        }
    },
});

    埋点配置

    // trackevent.config.js
import Vue from 'vue';
import {
  baidutongji
} from '../package.json'; // 获取百度应用id

// 注入百度统计上报功能
Vue.prototype.$baidutongji = {
  push() {
    // 只有正式环境才上报数据
    if ('_hmt' in window && window._hmt.push instanceof Function && arguments[0] instanceof Array) {
      try {
        window._hmt.push(arguments[0]); // 调用 SDK
      } catch (error) {
        console.log(error);
      }
    }
    // 开发环境直接输入埋点数据
    if (process.env.NODE_ENV !== 'production') {
      console.log(arguments[0]);
    }
  },
  trackevent() {
    if (arguments[0] instanceof Array) {
      const messages = ['_trackEvent'].concat(arguments[0]); // 添加埋点基础信息
      try {
        // 判断是否在全局下注册了vue实例，有的话就从状态管理器中取出数据
        if (window.vueApplication) {
          messages.push(window.vueApplication.$store.state.userinfo.real_name); // 添加用户信息
        }
      } catch (error) {
        console.log(error);
      }
      this.push(messages);
    }
  },
  trackpageview() {
    if (arguments[0] instanceof Array) {
      this.push(['_trackPageview'].concat(arguments[0])); // 添加埋点基础信息
    }
  }
};

export default {
  init() {
    if (process.env.NODE_ENV === 'production') {
      window._hmt = window._hmt || []; // 挂载到window上
      const hm = document.createElement('script');
      hm.src = `https://hm.baidu.com/hm.js?${baidutongji.key}`;
      const s = document.getElementsByTagName('script')[0];
      s.parentNode.insertBefore(hm, s);
    }
  },
  callback(data) {
    if (data instanceof Array) {
      Vue.prototype.$baidutongji.trackevent(data); // 上报事件分析数据
    }
  },
}

    vue实例化

    // main.js
import Vue from 'vue';
import ElementUI from 'element-ui';
import VueRouter from 'vue-router';
import VTrackevent from 'vue-trackevent';

import routerConfig from './router.config';
import trackeventConfig from './trackevent.config';

// 配置数据埋点
Vue.use(VTrackevent, trackeventConfig);

Vue.use(ElementUI);
Vue.use(VueRouter);
Vue.use(VueLazyload);

// 处理路由双击问题
const originalPush = VueRouter.prototype.push;
VueRouter.prototype.push = function push(location) {
  return originalPush.call(this, location).catch(err => err)
}

// 路由配置
const router = new VueRouter({
  routes: routerConfig.routes,
});
router.beforeEach((to, from, next) => {
  // vue SPA页面特殊处理
  if (to.path && router.mode === 'hash') {
    Vue.prototype.$baidutongji.trackpageview([`/#${to.fullPath}`]); // 发送页面 PV
  }
  next();
});

// 组件配置(支持目录写法，避免结构混乱)
const pluginFiles = require.context('./plugins', true, /.vue$/);
pluginFiles.keys().forEach(key => {
  const component = pluginFiles(key).default;
  if (component.name) {
    Vue.component(component.name, component);
  }
});

// 实例化
window.vueApplication = new Vue({
  el: '#app',
  router,
  template: '<router-view />',
  mounted() {
    const colorfulLog = (label = '', value = '', labelColor = '#35495e', valueColor = '#41b883') => {
      console.log(
        `%c ${label} %c ${value} %c`,
        `background:${labelColor} ; padding: 1px; border-radius: 3px 0 0 3px; color: #fff`,
        `background:${valueColor} ; padding: 1px; border-radius: 0 3px 3px 0; color: #fff`,
        'background:transparent;'
      );
    };
    colorfulLog('构建版本', process.env.CODE_VERSION);
  },
});

    可以看到我们使用了vue-trackevent模块，在实例化时传入了埋点配置，配置中向vue实例上注入了Vue.prototype.$baidutongji对象

    在页面跳转时调用了上报pv的方法 Vue.prototype.$baidutongji.trackpageview

    下面是上报事件的用法

    page

    <template>
  <div>
    <el-button v-trackevent="['hello','trackevent']">自动上报</el-button>
    <el-button @click="upload">人为上报</el-button>
  </div>
</template>

<script>
export default {
  methods: {
    upload() {
      this.$baidutongji.trackevent(["hello", "world"]); // 埋点
    }
  }
};
</script>
