# Apollo 环境变量

配合单点系统，适用于各种业务场景，建议子应用使用 node-sso-connect 模块

node-sso-connect

# 背 景

随着程序功能的日益复杂，程序的配置日益增多：各种功能的开关、参数的配置、服务器的地址等，这对程序配置的期望值也越来越高，在这样的大环境下，传统的通过配置文件、数据库等方式已经越来越无法满足开发人员对配置管理的需求。

配置中心应运而生。

# 创建流程

配置中心的最小粒度为 key ，每一个 应用 下都有一个唯一的 key ,系统以类似键值对的形式存储各个应用的配置。

# 创建应用

在使用配置中心前，需要先创建应用。在 /admin/application 应用管理页面点击 新增 按钮，并填写相关内容

字段 描述
APPID 应用唯一标识，需要妥善保管
应用名称 应用名称
私钥 用于做数据校验，需要妥善保管
ICON 应用 logo

# 创建环境

每一个应用都有一个专属自己的环境，在 /admin/config 环境管理页面可以看到刚刚创建的应用，点击进去就可以创建自己的参数。参数主要包括以下信息：

字段 描述
key 环境变量
value 值
备注 备注信息

# 发布流程

创建/编辑环境变量后，子应用并不会收到更新推送，可以在修改完一批后点击顶部的发布按钮，此时已接入的子应用会接收到当前所属应用的环境参数推送。

# 接入指南

子应用通过 node-sso-connect 模块快速接入，假设 sso 服务运行与 sso.test.com 站点，连接器端口默认为 3001，应用标识为：appid、私钥为：appsecret

// nodejs
// your http server code

import nsc from 'node-sso-connect';

nsc.connect({
	url: 'ws://sso.test.com:3001',
	appid: 'appid',
	appsecret: 'appsecret',
	onPublish: params => {
		console.log(params);
	},
});

其中 onPublish 回调为当前应用的环境变量，在连接器初始化成功时或 sso 主动发布时均会被触发。

和单点注销接入方法一致
