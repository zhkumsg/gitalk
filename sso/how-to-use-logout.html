# 单点注销

单点登录自然也会有单点注销，在一个子系统中注销，所有子系统的会话都将被销毁。

# 背 景

接入单点登录后，子应用通过接口可以获取到用户信息，一般情况下子应用会自行维持登录态，即只有登录行为在 sso 完成，其它业务直接从缓存中读取用户信息，不需要考虑其它子应用的登录态，也不受其它子应用影响。

然而在一些业务场景下，需要多个子应用配合使用，如微前端，往往有多个应用，例如：商品项目、订单项目、购物车项目，这些项目抽离出去单独开发，共享同一个用户信息，这时候就需要统一登录态了。

# 接入指南

接入分为两个步骤，分别是前端修改和后端修改

# 前端修改

在需要进行单点注销（统一退出）的时候，需要浏览器重定向到 sso 的注销页面，可以直接修改 url 重定向，也可以通过 iframe 的形式进行后台退出，推荐使用后者。

sso 的注销页面默认为：

http://sso.test.com:3000/logout/${appid}/${callback_url}

其中 appid 和 callback_url 和登录时类似

参数 说明
appid 应用标识, 在应用类别中可查看
callback_url 回调地址，经过 UrlEncode 编码后能正确访问的 url 地址

成功注销后默认重定向到登录页面。

如果使用 iframe 的形式，完成注销后会执行父窗口的 logout 的方法,如果自定义注销后的逻辑，可添加此方法

// 内部实现
if (window.parent.logout) {
	try {
		window.parent.logout(res);
	} catch (error) {
		console.log(error);
	}
}

# 后端修改

如果希望能接收单点注销的通知，只做前端修改是不够的，还需要在后端做相应的处理。

为了方便开发，我们提供了连接器模块，使用该模块可以实时接收到单点注销通知，下面是使用方法

# Installation

$ npm install node-sso-connect --save

# 使 用

// nodejs
// your http server code

import nsc from 'node-sso-connect';

// 连接sso服务
nsc.connect({
	url: 'ws://localhost:3001',
	appid: 'appid',
	appsecret: 'appsecret',
	onLogout: ssokeys => {
		console.log(ssokeys);
	},
});

但子应用（自己或其他子应用）注销时，注册了连接的应用都会收到注销通知，通过 ssokeys 即可判断是哪个用户注销了

ssokeys 的主要结构是{appid，key}，其中 appid 是应用 id，key 是单点登录时的回调 key，在登录重定向时可获取。

和 Apollo 环境变量接入方法一致
