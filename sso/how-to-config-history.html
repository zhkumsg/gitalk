# History 路由

本系统开发环境使用的路由模式的 history，所以需要做一些特殊处理

举个例子，刷新时，/admin/user对浏览器来说，是认为去请求/admin/user/index.html,但很明显，我们不存在这个文件，会抛 404 异常。一般情况下我们需要 nginx 做重定向配合，但这里使用原生 nodejs，不用 nginx，怎么处理呢？

其实也很简单，我们做一个中间件拦截 404，检测到 404 时，直接把index.html的内容返回去就可以了，这样就可以理解成不存在的文件统一执行入口文件，具体写法如下：

前端路由配置成 history 模式

// src/main.js
const router = new VueRouter({
	mode: process.env.NODE_ENV === 'production' ? 'history' : 'hash',
	routes: [
		{ path: '/login', component: Login },
		{
			path: '/admin',
			component: Admin,
			children: [{ path: 'user', component: User }],
		},
		{ path: '*', redirect: '/login' },
	],
});

kao 添加 404 中间件

// app.js   vue history兼容
app.use(async ctx => {
	if (ctx.response.status === 404) {
		const html = await new NotFoundCtrl().redirect();
		ctx.body = html;
	}
});

自动返回 index.html 信息

// NotFoundCtrl
const fs = require('fs');
const path = require('path');

module.exports = class NotFoundCtrl {
	/**
	 * 404 兼容vue
	 */
	redirect() {
		return new Promise(resolve => {
			const filepath = path.join(__dirname, '../dist/index.html');
			fs.readFile(filepath, 'utf8', (err, data) => {
				if (err) {
					resolve(`<pre>${err}</pre>`);
				} else {
					resolve(data);
				}
			});
		});
	}
};
